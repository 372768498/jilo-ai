## 一、技术方案对比

### 方案 A：RPA 自动化（Playwright）⭐ 推荐

| 维度 | 详情 |
|------|------|
| **工具** | `social-auto-upload`（已集成在 clawd skill 中）+ Playwright |
| **原理** | 模拟真实浏览器操作，填写标题、正文、上传图片、点击发布 |
| **优点** | ✅ 最成熟方案，GitHub 4.6k+ stars<br>✅ 我们已有现成 skill（`skills/xiaohongshu/`）<br>✅ 支持图文笔记 + 定时发布<br>✅ 无需逆向签名算法 |
| **缺点** | ⚠️ 需要首次扫码登录获取 Cookie<br>⚠️ Cookie 有效期有限（通常 7-30 天需刷新）<br>⚠️ 平台 UI 变更可能导致元素定位失败 |
| **成熟度** | ⭐⭐⭐⭐⭐ 生产可用 |

**我们已有的能力：**
- 渲染引擎：Markdown → 小红书风格图片卡片（1080×1440px，7种样式）
- 发布模块：基于 `social-auto-upload` 的 Playwright 自动上传
- Cookie 管理：扫码登录 → JSON 持久化

### 方案 B：xhs Python 库（API 模拟）

| 维度 | 详情 |
|------|------|
| **工具** | `ReaJason/xhs`（GitHub 2k+ stars）|
| **原理** | 通过 Playwright 获取 x-s 签名，然后直接调用小红书 Web API |
| **优点** | ✅ 比纯 RPA 更快更稳定<br>✅ 支持创建笔记、获取数据等 API<br>✅ Docker 部署签名服务 |
| **缺点** | ⚠️ 签名算法频繁更新，需跟进维护<br>⚠️ 需要 cookie 中 a1、web_session、webId 字段<br>⚠️ 存在法律灰色地带 |
| **成熟度** | ⭐⭐⭐⭐ 可用但需维护 |

### 方案 C：纯手动发帖

| 维度 | 详情 |
|------|------|
| **优点** | ✅ 零封号风险<br>✅ 无技术维护成本 |
| **缺点** | ❌ 不可扩展<br>❌ 每篇 5-10 分钟，日更 2-3 篇 = 每天 30 分钟 |
| **适合** | 初期测试阶段验证内容方向 |

### 方案 D：MediaCrawler（爬虫 + 数据分析）

| 维度 | 详情 |
|------|------|
| **工具** | `NanmiCoder/MediaCrawler`（热门开源爬虫）|
| **用途** | 爬取竞品笔记、分析热门内容、获取评论数据 |
| **定位** | **辅助工具**，不直接发帖，但对内容策略制定非常有价值 |

### 📌 推荐组合

```
内容创作：AI（Clawdbot）生成文案
      ↓
图片渲染：已有 render 脚本（1080×1440px）
      ↓
自动发布：social-auto-upload（RPA）
      ↓
数据分析：MediaCrawler 爬取竞品数据
      ↓
策略优化：根据数据调整内容方向
```

## 二、内容策略

### 2.1 最火内容类型（AI 工具方向）

#### 🔥 Tier 1：高互动率内容

1. **AI 工具合集帖**（收藏量最高）
   - 「2026 年必装的 10 个 AI 工具，第 5 个太绝了」
   - 「打工人效率翻倍！8 个免费 AI 神器推荐」
   - 「设计师必备 AI 工具清单，省下几万块外包费」

2. **对比测评帖**（讨论度最高）
   - 「ChatGPT vs Claude vs Gemini，实测对比谁更强」
   - 「5 个 AI 写作工具横评，哪个最适合小红书创作」

3. **教程干货帖**（转发量最高）
   - 「手把手教你用 AI 做 PPT，10 分钟搞定年终汇报」
   - 「AI 修图神器！一键去背景、换风格、修瑕疵」

#### ⭐ Tier 2：稳定流量内容

4. **场景化推荐**
   - 「学生党必备 AI 工具」「自媒体人 AI 工具箱」「程序员 AI 效率工具」

5. **新工具首发/体验**
   - 「刚发现一个宝藏 AI 工具，竟然完全免费！」

6. **AI 行业资讯**
   - 「OpenAI 又出新功能了！这次真的很实用」

#### 💡 Tier 3：长尾内容

7. **问题解决型**
   - 「还在手动做表格？这个 AI 工具帮你 1 秒搞定」

8. **个人体验分享**
   - 「用了一个月 AI 工具后，我的工作效率变化」

### 2.2 内容风格要点

| 要素 | 建议 |
|------|------|
| **标题** | 数字 + 情绪词 + 具体场景（「5个」「绝了」「打工人必看」）|
| **封面** | 大字标题 + 工具截图拼图，色彩鲜艳（推荐紫/橙/蓝色调）|
| **正文** | 每段 2-3 行，多用 emoji，重点加粗，段落间空行 |
| **图片数量** | 6-9 张为佳（合集帖用卡片式图片）|
| **话题标签** | 5-10 个，混合大话题 + 精准话题 |
| **引导语** | 「关注我获取更多 AI 工具推荐」「收藏 = 学到了」|

### 2.3 话题标签模板

```
#AI工具 #AI推荐 #效率工具 #打工人必备 #学习工具
#人工智能 #ChatGPT #科技好物 #数码推荐 #宝藏工具
#jilo（品牌词，初期可不加）
```

## 六、风险提示

### 6.1 封号风险 🚨

| 风险行为 | 严重程度 | 后果 |
|----------|----------|------|
| 短时间大量发帖（>5篇/天）| 🔴 高 | 限流 → 禁言 → 封号 |
| 内容高度重复/洗稿 | 🔴 高 | 笔记被删 → 账号降权 |
| 正文带外链 | 🟡 中 | 笔记限流 |
| 评论区引流过于明显 | 🟡 中 | 评论被删 → 禁言 |
| 频繁切换 IP/设备 | 🟡 中 | 触发风控验证 |
| 使用自动化工具 | 🟡 中 | 被检测则封号 |
| Cookie 过期后强制操作 | 🟢 低 | 操作失败 |

### 6.2 自动化安全措施

```markdown
✅ 必做：
1. 发帖间隔 > 30 秒（建议 2-5 分钟）
2. 每天发帖不超过 3 篇
3. 使用 stealth.js 绕过浏览器检测
4. 模拟真实操作节奏（随机延迟）
5. 定期手动登录，保持账号活跃
6. 每篇内容必须原创，不可模板化太重

❌ 禁止：
1. 不要批量注册账号
2. 不要用同一设备登录多个账号
3. 不要在凌晨自动发帖（不符合真人行为）
4. 不要复制粘贴相同内容
5. 不要使用非法获取的 Cookie
```

### 6.3 法律风险

- **爬虫相关**：中国已有多起爬虫违法案例，MediaCrawler 仅用于学习和轻量数据分析
- **自动化发帖**：属于灰色地带，遵守平台规则、不滥用是关键
- **内容合规**：AI 工具推荐内容需确保真实，不虚假宣传

### 6.4 备用方案

如果自动化方案被检测：
1. **降级为半自动**：AI 生成内容 + 手动发布
2. **使用手机端发布**：更不容易被检测
3. **外包运营**：找小红书运营人员代发
