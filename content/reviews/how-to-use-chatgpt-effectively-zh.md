## 入门：注册与设置

### 第一步：创建账户

1. 访问 [chat.openai.com](https://chat.openai.com)
2. 点击 **Sign Up（注册）**
3. 使用邮箱、Google 账号、Microsoft 账号或 Apple ID 注册
4. 验证邮箱地址
5. 完成手机号验证（需要海外手机号）

> 🔔 **国内注册提示：** 注册需要海外手机号接收验证码。可以使用虚拟手机号服务，或请海外朋友协助。部分第三方平台提供 ChatGPT 的镜像服务，但请注意数据安全。

### 第二步：选择订阅方案

ChatGPT 提供以下方案：

| 方案 | 价格 | 功能特点 |
|------|------|----------|
| **免费版** | $0/月 | GPT-4o mini、有限的 GPT-4o 访问 |
| **Plus** | $20/月 | 完整 GPT-4o、GPT-4、DALL·E、高级数据分析、自定义 GPTs |
| **Team** | $25/人/月 | Plus 全部功能 + 工作空间管理、更高限额 |
| **Enterprise** | 定制价格 | 无限使用、管理控制台、SOC 2 合规 |

**建议：** 先用免费版学习基本操作。当你发现使用频率很高或需要 GPT-4 级别的复杂推理时，再升级到 Plus。

### 第三步：配置个人设置

在开始第一次对话前，优化你的使用体验：

- **自定义指令**：进入设置 → 个性化 → 自定义指令。告诉 ChatGPT 你的身份信息和你希望它如何回复。
- **记忆功能**：开启记忆功能，让 ChatGPT 在不同对话间记住你的偏好。
- **数据控制**：在设置 → 数据控制中查看哪些数据会被用于模型训练。

## 了解 ChatGPT 的界面

### 聊天窗口

主界面非常直观：

- **消息输入框**：在底部输入你的提示词
- **模型选择器**：在聊天顶部选择 GPT-4o、GPT-4 或 GPT-4o mini
- **对话历史**：左侧边栏列出所有历史对话
- **附件功能**：点击回形针图标上传文件、图片或文档
- **语音模式**：点击耳机图标开启语音对话

### 核心概念

- **对话上下文**：ChatGPT 会记住单次对话中的所有内容。切换话题时，建议开启新对话。
- **Token 限制**：每个模型有上下文窗口限制（如 GPT-4o 支持 128K tokens）。过长的对话可能丢失早期上下文。
- **温度参数（Temperature）**：控制回答的随机性，默认设置在创造性和准确性之间取得平衡。

## 基础对话技巧

### 具体化，不要模糊

高效使用 ChatGPT 最重要的一条规则是：**具体、具体、再具体。**

❌ **模糊提示：**
```
跟我讲讲营销。
```

✅ **具体提示：**
```
请介绍 5 种适合 B2B SaaS 初创公司的数字营销策略，预算为每月 3 万元人民币，
重点关注 LinkedIn 和内容营销渠道。
```

### 提供背景信息

给 ChatGPT 足够的上下文，效果会显著提升：

```
我是一名专注于 React 的自由前端开发者。我的客户需要一个集成了 Stripe 支付的
电商网站。请帮我规划项目结构和需要的核心组件。
```

### 指定输出格式

明确告诉 ChatGPT 你想要什么格式的输出：

```
请制作一个对比表格，比较 5 款主流项目管理工具（Asana、Monday、Trello、Jira、
飞书项目），包含以下列：价格、最适合的场景、核心功能、集成能力、不足之处。
```

### 迭代优化

不要满足于第一次回答。用后续消息持续优化：

- "请更简洁一些"
- "增加更多技术细节"
- "用非技术人员能理解的语言重写"
- "给我 3 个不同版本"

## 高级提示词策略

### 1. 角色扮演法

给 ChatGPT 指定一个具体角色，获得专家级回答：

```
你是一位拥有 15 年经验的资深数据科学家，专精机器学习。请向我解释梯度提升
（Gradient Boosting）算法。我是一名了解基础统计学但从未建过 ML 模型的初级
开发者。
```

### 2. 思维链（Chain of Thought）

让 ChatGPT 逐步推理，处理复杂问题：

```
请一步一步解决这个问题，展示每步的推理过程：

某公司 Q1 营收增长 15%，Q2 下滑 8%，Q3 增长 22%，Q4 增长 5%。
如果 Q1 初始营收为 100 万元，请计算最终年营收是多少？
同时计算年复合增长率（CAGR）。
```

### 3. 少样本提示（Few-Shot Prompting）

提供示例来引导输出格式：

```
将以下产品描述转换为简洁有力的一句话广告语：

产品：防水蓝牙音箱，续航 20 小时
广告语："你的音乐，无处不在。风雨无阻，从晨到暮。"

产品：AI 驱动的学术论文写作助手
广告语："从初稿到发表——你的研究值得懂学术的 AI。"

产品：支持人脸识别的智能家庭安防摄像头
广告语：
```

### 4. 结构化输出请求

要求 JSON、CSV 或其他结构化格式：

```
分析以下客户反馈，并返回一个 JSON 对象，包含：
- "sentiment"：正面/负面/中性
- "topics"：提到的主要话题数组
- "urgency"：紧急程度（1-5 分）
- "suggested_action"：建议的下一步行动

反馈："我已经等了 3 周了，订单还没到。客服每次都说'在处理中'。
太荒谬了，我要退款。"
```

### 5. 系统级指令

在使用 API 或自定义 GPTs 时，系统消息是基础：

```
System: 你是一位简洁的技术文档写作者。始终遵守以下规则：
1. 使用主动语态
2. 句子不超过 20 个字
3. 每个概念都配上代码示例
4. 使用 Markdown 格式输出
5. 每个章节末尾附上"要点总结"
```

### 6. 约束式提示

设置明确的边界来聚焦输出：

```
编写一个 Python 函数，要求：
- 输入：字典列表
- 功能：按指定键值对过滤
- 输出：排序后的结果
- 只使用内置库（不用 pandas/numpy）
- 包含类型注解和文档字符串
- 代码不超过 20 行
```

## 自定义 GPTs：打造你的专属 AI 助手

### 什么是自定义 GPTs？

自定义 GPTs 让你创建专门化的 ChatGPT 实例，包含：
- 预设指令
- 上传的知识文件
- 自定义操作（API 调用）
- 特定的对话开场白

### 创建你的第一个自定义 GPT

1. 访问 [chat.openai.com/gpts/editor](https://chat.openai.com/gpts/editor)
2. 点击 **Create（创建）**
3. 使用 **Configure（配置）** 标签页设置：

```
名称：SEO 博客写手
描述：创建 SEO 优化的博客文章，包含规范的标题结构、元描述和关键词整合。

指令：
你是一位专业的 SEO 内容写作者。当收到主题和目标关键词时：
1. 生成一个包含关键词的吸引眼球的标题
2. 撰写元描述（150-160 个字符）
3. 编写包含 H2/H3 标题的完整文章
4. 自然地整合关键词 3-5 次
5. 提供内链建议
6. 添加适合 Schema 标记的 FAQ 部分
7. 推荐 3 篇相关文章主题

始终使用对话式、引人入胜的语气。段落要短。
包含可操作的建议，而不仅仅是理论。
```

4. 上传相关知识文件（品牌风格指南、写作规范等）
5. 设置对话开场白，如"写一篇关于 [主题] 的博客"

## 使用 OpenAI API

### 获取 API 密钥

1. 访问 [platform.openai.com](https://platform.openai.com)
2. 进入 **API Keys** 页面
3. 点击 **Create new secret key**
4. 安全保存密钥（之后无法再次查看）

> 💡 **提示：** API 使用需要绑定海外信用卡进行充值。国内用户可以考虑使用虚拟信用卡服务。

### 第一次 API 调用

```python
from openai import OpenAI

client = OpenAI(api_key="你的-api-key")

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一位专业的编程助手。"},
        {"role": "user", "content": "写一个 Python 函数，使用正则表达式验证邮箱地址。"}
    ],
    temperature=0.7,
    max_tokens=1000
)

print(response.choices[0].message.content)
```

### 关键 API 参数

| 参数 | 说明 | 常用值 |
|------|------|--------|
| `model` | 使用的模型 | `gpt-4o`、`gpt-4o-mini`、`gpt-4-turbo` |
| `temperature` | 随机性（0=确定性，2=高创造性） | 大多数任务用 0.3-0.7 |
| `max_tokens` | 最大回复长度 | 根据任务设置 500-4000 |
| `top_p` | 核采样（temperature 的替代方案） | 0.9-1.0 |
| `frequency_penalty` | 减少重复 | 0.0-0.5 |
| `presence_penalty` | 鼓励新话题 | 0.0-0.5 |

### 流式响应

在应用中实现实时输出：

```python
stream = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "解释量子计算"}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")
```

### 函数调用（Function Calling）

让 ChatGPT 触发应用中的函数：

```python
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {"type": "string", "description": "城市名称"},
                    "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
                },
                "required": ["location"]
            }
        }
    }
]

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "东京现在天气怎么样？"}],
    tools=tools,
    tool_choice="auto"
)
```

## 10 个即用型 Prompt 模板

### 1. 内容写作

```
请写一篇关于 [主题] 的 [类型：博客/文章/简报]。

目标读者：[描述读者群体]
语气：[专业/轻松/学术/幽默]
字数：[目标字数]
需包含：[具体元素，如数据、案例、行动号召]

结构要求：
- 吸引眼球的开头
- 清晰的二级标题
- 实用、可操作的建议
- 有力的结尾并附上下一步行动
```

### 2. 代码生成

```
用 [编程语言] 编写 [描述功能] 的代码。

要求：
- 输入：[描述输入格式]
- 输出：[描述期望输出]
- 错误处理：[指定错误情况]
- 性能要求：[任何限制]

需包含：
- 类型注解
- 文档字符串/注释
- 单元测试（至少 3 个测试用例）
- 使用示例
```

### 3. 数据分析

```
分析以下数据并提供：

1. 摘要统计信息
2. 关键趋势和模式
3. 异常值或离群点
4. 3 条可执行的洞察
5. 可视化建议

数据：
[粘贴你的数据]

背景：[这些数据代表什么，基于它需要做什么决策]
```

### 4. 翻译与本地化

```
将以下文本从 [源语言] 翻译为 [目标语言]。

要求：
- 保持原文的语气和风格
- 适当调整文化相关的表达以适应目标受众
- 确保专业术语准确（必要时在括号内保留原文）
- 保留格式（标题、项目符号等）

文本：
[粘贴文本]
```

### 5. 学习与概念解释

```
使用费曼技巧解释 [概念]：

1. 用简单的话解释（仿佛在教一个 12 岁的孩子）
2. 找出简单解释中的不足之处
3. 深入解释每个不足
4. 用日常生活中的类比来说明
5. 给出一个实际应用的例子

我目前的理解：[描述你已知的内容]
我的目标：[你想用这个知识做什么]
```

### 6. 邮件撰写

```
写一封 [类型：陌生拜访/跟进/投诉/感谢] 邮件。

背景：[情况说明]
发件人：[你的角色]
收件人：[对方的角色和与你的关系]
目标：[你想达到什么效果]
语气：[专业/友好/坚定/致歉]
长度：[简短/适中/详细]

需要涵盖的要点：
- [要点 1]
- [要点 2]
- [要点 3]
```

### 7. 商业策略

```
请扮演一位资深商业顾问，分析这个业务挑战：

行业：[你的行业]
公司规模：[员工数/营收]
挑战：[描述问题]
限制条件：[预算、时间、资源]
已尝试的方法：[你做过什么]

请提供：
1. 根因分析
2. 3 个战略方案，附优缺点
3. 推荐的行动计划和时间表
4. 关键跟踪指标
5. 潜在风险和应对策略
```

### 8. 简历优化

```
请优化我的简历要点，使其更有影响力。

当前职位：[你的头衔]
目标职位：[期望职位]
目标行业：[行业]

使用 STAR 方法（情境、任务、行动、结果）改写以下描述，
尽可能加入可量化的数据：

[粘贴你当前的简历要点]
```

### 9. 社交媒体内容

```
为 [平台：LinkedIn/微博/小红书/公众号] 创建 [时间段] 的内容日历。

品牌：[描述你的品牌/个人品牌]
目标：[知名度/互动/线索/思想领导力]
内容主题：[列出 3-5 个内容支柱]
发布频率：[多久发一次]

每条内容请提供：
- 开头钩子
- 完整文案
- 话题标签建议
- 最佳发布时间
- 互动引导（问题或行动号召）
```

### 10. 会议与文档总结

```
用三种格式总结以下 [会议记录/文档/文章]：

1. **高管摘要**（3-5 句话，给领导看）
2. **关键要点**（要点列表）
3. **行动项**（谁需要做什么，截止日期）

同时识别出：
- 已做出的决策
- 待解决的问题
- 提出的风险或顾虑

内容：
[粘贴你的内容]
```

## 提升效果的实用技巧

### 1. 善用"角色扮演"框架
用角色开头："你是一位资深的 DevOps 工程师……" 这会让 ChatGPT 调动特定领域的知识。

### 2. 将复杂任务拆分为步骤
不要用一个巨大的提示词，而是使用多轮对话：
1. 第一条消息：定义项目，获取大纲
2. 第二条消息：逐章节展开
3. 第三条消息：审查和优化

### 3. 让 ChatGPT 先问你问题
```
我想为我的创业公司制定一个营销计划。在开始之前，请先问我 10 个问题，
这些问题能帮助你制定出最有效的方案。
```

### 4. 要求"展示推理过程"

## 常见问题

### 1. ChatGPT 是免费的吗？

是的，ChatGPT 提供免费版，可以使用 GPT-4o mini 和有限的 GPT-4o 访问。如果需要无限的 GPT-4o 访问、DALL·E 图像生成、自定义 GPTs 和高级数据分析等功能，需要订阅 ChatGPT Plus（$20/月）。

### 2. ChatGPT 的回答准确吗？我可以完全信任它吗？

ChatGPT 能力很强，但并非万无一失。它可能会生成听起来可信但实际错误的信息（"幻觉"现象）。对于关键事实，特别是涉及医疗、法律或金融的决策，务必进行验证。把它当作起草和头脑风暴工具，然后对重要信息进行事实核查。

### 3. ChatGPT 会记住之前的对话吗？

在单次对话中，ChatGPT 会维持完整的上下文记忆。在不同对话之间，它通过记忆功能（如果开启的话）来记住你分享过的关键偏好和信息。你可以在设置 → 个性化 → 记忆中查看和管理这些记忆。

### 4. GPT-4o 和 GPT-4o mini 有什么区别？

GPT-4o 是 OpenAI 的旗舰模型——更快、更智能，在复杂推理、编程和细微任务上表现更出色。GPT-4o mini 是更小、更快、更便宜的模型，适合简单任务如起草文本、总结和基础问答。可以把它们理解为资深专家和能干的初级员工的区别。

### 5. 我可以将 ChatGPT 用于商业目的吗？

可以。根据 OpenAI 的服务条款，你拥有 ChatGPT 生成的内容的所有权，可以用于商业目的，包括内容创作、产品开发和客户服务。但你应该审查输出的准确性，并确保不侵犯现有版权。
